# 打造移动端适配的高级表格组件

## 表格组件：需求分析&虚拟无限滚动原理

了解EP的表格组件：el-table

了解EP的虚拟化表格组件：el-table-v2。优点性能好，缺点功能少，且还有很多未修复的bug，移动端适配有问题。

优先使用el-table，如果需要优化性能或移动端适配，考虑el-table-v2。

为VTable添加相关的TS定义

- 为VTableProps添加pagination属性，类型为`PaginationType`
- 为VTableProps添加columns属性，类型为`TableColumnCtx<any>`

将VTable的相关TS定义转移到单独的.d.ts，方便其他组件导入

在VTable中定义props，并与模板进行绑定，替换测试数据

在table.vue中添加测试数据，测试VTable组件的展示效果，必要时为部分属性添加默认值

深入研究VTable组件的默认样式错乱问题

- 原因：vue-sfc中，defineProps会将未传入的属性设置默认值，这与EP官方的默认值不一致。布尔类型默认值为false，其他类型默认值为undefined。
- 解决思路：类型宽 + 运行时窄
- 解决方案1：使用`/*@vue-ignore*/`魔法注释，跳过extends TableProps。使用`$attr`属性透传，传递EP组件的内部属性
- 解决方案2：使用index.ts统一导出组件，导出时使用宽类型定义，内部defineProps时使用窄类型定义



## 进阶表格-灵活的Slot传参-兼容column&table

按照EP官方文档，逐个测试VTable的透传功能，包括斑马纹、边框、状态、溢出工具提示等

使用table插槽传递column插槽数据（以固定列为例）

- 使用default插槽传递固定列配置，`<template>` + `<slot>` 实现插槽透传
- 在固定列上添加Detail和Edit按钮，添加事件，并通过插槽的scope获取当前行的信息
- 在table.vue中添加测试数据，测试table插槽功能
- 注意：default插槽中可设置多个列

使用table-column插槽传递column插槽数据

- 在table-column的类型定义TableColumnType中，扩展属性，添加defaultSlot和headerSlot，slot本身会传递一个Component
- 在循环columns时，设置当前列的default和header插槽信息，通过v-slot:xxx="scope"设置插槽props
- 使用动态组件`<component>`接收插槽内容，通过v-bind="scope"接收props数据
- 在table.vue中添加测试数据，测试table-column插槽功能

优化pagination展示时机，在父容器上使用v-if判断pagination信息是否传递

（可选）tsconfig.app.json中，关闭any类型的严格提示，`noImplicitAny`设置为false
