## 打包优化：分模式打包&打包命令参数传递

### 添加构建环境判断

在vite.config.ts中，将defineConfig的参数修改为函数，接收参数useconfig，解构出mode

设置变量isProd = mode === ‘production’，mode通常为 `development` 或 `production`

isProd 为 `false` 时，才使用开发模式相关的插件，例如：ElementPlusResolver

### 添加环境变量

在`.env` 文件中添加环境变量，不同模式下使用不同的变量值

.env                  # 所有情况下都会加载 

.env.[mode]    # 只在指定模式下加载 

变量名必须以 `VITE_` 开头

### 添加特定情况下的变量

并非开发或生产环境，而是只在特定环境下添加变量。

安装 cross-env 库，用于跨平台设置环境变量

如下，只在执行analysis命令时，才添加变量ANALYSIS

```json
"analysis": "cross-env ANALYSIS=true npm run p"
```

通过`process.env.ANALYSIS`获取设置的环境变量

`process.env.ANALYSIS`为`'true'`时，visualizer插件的`open`配置才为true，才会进行包体分析

## echarts打包优化：如何定位与解决生产代码报错

捕获程序错误。在main.ts中，添加app.config.errorHandler，可在浏览器打印捕获到的错误

定位错误位置。在在vite.config.ts中，设置build.sourcemap = true，重新build，可在浏览器显示错误的具体代码位置

分析错误原因。在VueEcharts.vue中，use方法在生产环境报错。生产环境使用的是cdn全量引入，可能不太兼容。

排除错误。在VueEcharts.vue中，为onBeforeMount钩子添加判断，非生产环境才执行。

配置优化。为 build.sourcemap 添加判断，特定debug环境才为true。



## 打包优化：选择不同的版本&设置script属性&切换CDN源

使用cdn时，选择包体的生产版本。设置path时，选择prod或min版本，代码被压缩过

使用cdn时，控制script的运行时机。设置transform函数，设置scriptNode的属性，与页面渲染无关的设置async，有关的设置defer

**注意：vite-plugin-cdn-import中配置的是generateScriptTag函数**

使用cdn时，选择CDN的源。设置url即可，例如可以设置为 `https://unpkg.com`

**注意：vite-plugin-cdn-import中配置的是prodUrl**

附加：解决components目录下的组件未被打包的问题

- 在rolldownOptions中再引入一次Components，解决打包时，components的组件没有被打包的问题
- `vite-plugin-cdn-import` 与 `unplugin-vue-components` 冲突导致
- 因为cdn引入时，components的组件没有被打包，所以需要再引入一次，让打包时，components的组件被打包
- issue参考：https://github.com/unplugin/unplugin-vue-components/issues/664
- issue参考：https://github.com/MMF-FE/vite-plugin-cdn-import/issues/54
